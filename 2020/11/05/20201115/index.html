배운점이 많은 하루

오늘 배운점에 대해 설명해볼수 있는 기회가 있었습니다. css의 margin-left와 padding에 대한 차이와, 순수 바닐라 js코드를 바탕으로 한 JSON 파일 파싱까지, 수많은것에 대해 알려줄수 있게 되었습니다.

처음엔 이해도 못했던 코드가, 이러한 방식으로 작동하게 되고 (데이터 가져오기 - 데이터 가공하기 - 데이터 추가로 가공하기 - 데이터 보기좋게 정렬하기)활용이 된다는 점이 인상깊었습니다.

어찌보면 Json을 가져올수 있는 간단한 소스코드 일수 있겠지만, 이점에서 제가 본받아야 할점들을 생각해 볼수 있었습니다.

첫번째는 데이터만 가져와서 끝나는 일이 아니였습니다.
데이터만 가져오고 바로 소스코드를 가져오려고 하니깐, 배열로만 불러올수 있게 되어서 이문제점에 대해 생각하게 되었습니다. 여기서 늘, 다른 예시를 찾아보고 해보면 될까란 생각을 했지만, 이러다간 또 기초를 놓칠거 같아 해답을 가질수 있는 키워드로 계속 찾아보게 되었습니다.

구글링을 통해, Json 데이터를 쉽게 가져올수 없을까란 키워드로만 검색 하게 되었고, 그에 대한 해답은 parse에 있었습니다.

parse의 데이터 변환을 통해 Json 데이터를 더욱더 깔금하게 불러올수 있게 되었고, 이를 바탕으로 해서 나중에도 이 데이터는 이 데이터다! 라는걸 각인하기 위해 조금더 이름에 대해 신경을 쓰게 되었습니다.

이러한 데이터 변경에 대한 구조를 통해, 조금더 이 코드에 대해 생각해볼수 있게 되었습니다.

추신, 중복된 코드를 사용한거 같습니다 음,, 어찌보면 잘못된 접근을 통해 들어온거 같네요. 하지만 획득한점이 있는거 같습니다. 너무 꼬인 코드로 너무 꼬이게 접근한거 같네요. 내일 수정이 필요해 보입니다.
